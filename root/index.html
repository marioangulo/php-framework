Users & Groups This is the user system (it's a big deal).

Users

You may grant users membership to groups. You may grant security permissions to users granularly to override the settings found in the users' group permissions.

[1] If a group has permission to a security setting then any user who is a current member of that group also has permission.
[2] If you need a user to be a member of a group but don't want the user to have a specific permission that the group does has permission to, you may deny access to the user individually from the user details > permissions screen.

User Groups

What you'll do most of the time is grant permissions to user groups. The users in a group will inherit the group's permission settings.

Group Categories

It also helps to organize your groups into categories. For example you could have an "employee" category and use the different groups in that category as the departments of a company.

Permission & Access How we let the right one's in.

Security Settings

Security settings are an important part of Footprint. Developers who work on the application will be the ones who put the security settings into place.

[1] Deleting or modifying these settings could cause irreversible damage or a lot of headaches for the developer. You've been warned.

IP Access

This area allows you to add trusted IP addresses to the application. All other IP addresses will be denied access (when using "require-session"). This is a great feature for organizations that don't want users logging in from outside a specific location.

[1] Don't forget about the "May Skip IP Check" security setting! Some web applications have users logging in from unknown locations, and that's ok.  Just grant this permission to groups or users who should skip IP address validation.

System Logs Keeping track of stuff that happened.

User History

Developers of the application install history log functions for certain actions and/or events users make. After installed they generated logs can be found here.

Errors

If error logging is turned on, you can view all errors that the system was able to catch. You can search, view, download and delete these error logs. See lots of debug information such as HTTP GET, HTTP POST, Session, Cookie and other environment data.
